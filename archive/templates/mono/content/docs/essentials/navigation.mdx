# Navigation

Your site's structure lives in `content/docs/docs.json`. It controls the top
tabs, the sidebar groups, and the API Reference.

## Groups and pages

The simplest config is a list of groups, each with pages:

```json docs.json
{
  "name": "My Docs",
  "navigation": [
    {
      "group": "Get Started",
      "pages": [
        { "title": "Introduction", "slug": "", "file": "index.mdx" },
        { "title": "Quickstart", "slug": "quickstart", "file": "quickstart.mdx" }
      ]
    }
  ]
}
```

<Note>
  `slug` is the URL path (an empty string is the index). `file` is the MDX file
  relative to `content/docs`. Nest pages with a `children` array.
</Note>

## Tabs

Add top-bar tabs to split Guides from your API Reference:

```json docs.json
{
  "tabs": [
    { "tab": "Guides", "navigation": [ /* groups */ ] },
    { "tab": "API Reference", "openapi": "openapi.json" }
  ]
}
```

## API Reference from OpenAPI

Point a tab's `openapi` field at a spec file (JSON or YAML) in `content/docs`.
The framework generates the entire API Reference — endpoint list, parameters,
request/response schemas, a cURL sample, and a live "Try it" playground — from
the spec.

<Tip>
  This is the standard: API docs come from OpenAPI, everything else is MDX. Keep
  your spec accurate and your reference stays accurate for free.
</Tip>

## Top bar

```json docs.json
{
  "navbarLinks": [{ "name": "Blog", "href": "/blog" }],
  "anchors": [{ "name": "GitHub", "href": "https://github.com/you/repo", "icon": "github" }],
  "cta": { "label": "Dashboard", "href": "https://app.example.com" }
}
```
