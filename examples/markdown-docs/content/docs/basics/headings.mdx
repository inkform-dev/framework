# Headings

Headings give your content structure. They create the hierarchy visible in the table of contents and are mapped to `<h1>` through `<h6>` in HTML.

## Syntax

Headings are written with `#` characters at the start of a line. The number of `#` signs determines the level.

<Tabs>
  <Tab title="Markdown">
    ```markdown
    # Heading 1

    ## Heading 2

    ### Heading 3

    #### Heading 4

    ##### Heading 5

    ###### Heading 6
    ```
  </Tab>
  <Tab title="Result">
    The rendered output maps to `<h1>` through `<h6>`. In a docs site, `h1` is typically the page title, `h2` and `h3` appear in the table of contents, and `h4`–`h6` are used for deeper nesting.
  </Tab>
</Tabs>

## Alternate syntax (Setext style)

CommonMark also supports an older "Setext" heading style using underlines. It only works for `h1` and `h2`:

```markdown
Heading 1
=========

Heading 2
---------
```

<Warning>
  Avoid the Setext style. It only supports two levels, it is harder to scan, and it is easy to accidentally create a heading when you meant a paragraph followed by a horizontal rule. The ATX style (`#`) is universally preferred.
</Warning>

## Best practices

<CardGroup cols={2}>
  <Card title="One h1 per page">
    Every page should have exactly one `# h1`. It is the page title. Using multiple `h1` headings is confusing for screen readers and SEO tools.
  </Card>
  <Card title="Do not skip levels">
    Go from `h2` to `h3`, not from `h2` to `h4`. Skipping levels breaks the document outline and can confuse accessibility tools.
  </Card>
  <Card title="Space after #">
    Always put a space between `#` and the heading text: `## My Section`. Some parsers require it; all accept it.
  </Card>
  <Card title="Keep headings short">
    Headings appear in tables of contents and anchor links. Long headings create awkward URLs like `#how-to-configure-the-database-connection-string`.
  </Card>
</CardGroup>

## Heading anchors

Every heading automatically gets an anchor ID derived from its text. The heading:

```markdown
## Getting Started
```

...becomes `<h2 id="getting-started">` in HTML, making it linkable as `page#getting-started`.

<Tip>
  In most Markdown renderers, you can link directly to a heading on the same page using `[Jump to section](#getting-started)`. The ID is generated by lowercasing the text and replacing spaces with hyphens.
</Tip>

## Closing `#` signs (optional)

You may optionally close an ATX heading with `#` signs. The following are equivalent:

```markdown
## My Section ##

## My Section
```

The closing `#` signs are stripped and have no effect on the heading level. They are a stylistic choice from early Markdown documents — you will rarely see them in modern writing.
