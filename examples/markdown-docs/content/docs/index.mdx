# Markdown Guide

Markdown is a lightweight markup language that converts plain text into structured HTML. Created by John Gruber in 2004, it has become the universal language of technical writing — from README files and documentation to blog posts and wikis.

## Why this guide exists

There are many Markdown tutorials online, but most stop at the basics. This guide goes further:

- It shows **source and result side by side** so you understand what every syntax element produces.
- It covers **GitHub-Flavored Markdown (GFM)**, the superset used by GitHub, GitLab, and most modern tools.
- It explains **MDX**, the format that powers this site, so you can build rich documentation.

<Note>
  New to Markdown? Start with [Why Markdown](/why-markdown) to understand what makes it so widely adopted, then move to [Headings](/basics/headings) to write your first content.
</Note>

## Quick look

Here is the essence of Markdown in ten lines:

```markdown
# Heading 1

This is a paragraph with **bold** and _italic_ text.

- Item one
- Item two

[Link text](https://example.com)

![Alt text](image.png)
```

That is it. The rest is detail — and that detail is what this guide covers.
